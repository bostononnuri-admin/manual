1장

시스템 개요

이 플랫폼은 교회에서 여는 각종 행사(예: VBS, 새가족 환영회, 청년부 모임, 봉사자 모집 등)의 참가자 및 봉사자 신청서를 온라인으로 만들고, 신청자가 직접 작성한 내용을 구글 스프레드시트에 자동으로 모아 주어 행사담당자가 관리할 수 있도록 도와주는 웹 도구입니다.

1-1. 이 플랫폼으로 할 수 있는 일
1
행사별 신청 페이지를 몇 분 만에 생성
행사 이름·날짜·모집 유형(참가자/봉사자/자유 유형)만 입력하면 독립된 신청 페이지가 자동으로 만들어집니다. 별도의 홈페이지 개발이나 디자인 작업이 필요 없습니다.
2
고유 링크와 큐알코드로 홍보
생성된 신청 페이지는 짧은 링크와 큐알코드 이미지로 즉시 공유할 수 있습니다. 주보·단체 대화방·게시판 어디서든 같은 페이지로 연결됩니다.
3
신청 내역 자동 저장 및 실시간 조회
신청자가 제출하면 즉시 구글 스프레드시트에 한 줄씩 저장됩니다. 행사 담당자는 해당 스프레드시트 편집 권한을 이메일로 받아 언제든 현황을 확인·관리할 수 있습니다.
4
신청 기간·정원 자동 관리
시작일 이전에는 "예정" 안내, 마감일 이후에는 "마감" 안내가 자동 표시됩니다. 유형별 정원을 지정하면 선착순으로 마감되며, 정원이 차면 그 유형의 신청만 자동으로 닫힙니다.
1-2. 사용하는 사람과 역할
가

신청자 (참가자·봉사자)

주보·대화방·게시판에서 공유된 링크 또는 큐알코드를 통해 신청 페이지에 접속합니다. 원하는 신청 유형을 고르고 이름·연락처 등 필요한 정보를 입력한 후 제출 버튼을 누르면 신청이 완료됩니다. 별도 로그인이나 회원가입은 필요하지 않습니다.
나

행사 담당자 (부서·사역팀 사역자, 팀장)

행사 생성 시 담당자 이메일로 등록된 사람입니다. 행사 신청서를 직접 생성하고, 관리 할 수 있습니다. 신청서가 만들어지면 해당 스프레드시트의 편집 권한이 자동으로 공유됩니다. 담당자는 신청 현황을 직접 구글 스프레드시트에서 확인하고, 신청자에게 안내 연락을 보내는 등의 후속 업무를 담당합니다.
다

시스템 관리자 (행정 간사)

플랫폼 전체를 뒤에서 뒷받침하는 사람입니다. 행사 담당자가 직접 신청서를 만들도록 권장하되, 요청이 있을 때는 대신 생성해 드립니다. 그 외에 생성된 전체 신청서 목록 관리, 구글 드라이브 데이터 백업, 시스템이 정상 동작하도록 유지, 그리고 행사 담당자가 사용에 어려움을 느낄 때 지원하는 것이 주된 역할입니다.
2장

전체 운영 흐름

행사 하나가 만들어져서 신청이 마감되기까지 일어나는 전체 과정을 순서대로 보여드립니다. 행사 담당자가 직접 신청서를 만들고 운영하며, 시스템 관리자(행정 간사)는 뒤에서 데이터 백업·시스템 유지·어려움 발생 시 지원을 담당합니다.

1
행사 담당자가 직접 행사 신청서 생성
행사 이름·기간·담당 부서·4자리 비밀번호·담당자 이메일·모집 유형을 입력하고 "생성" 버튼을 누릅니다.
▼
2
스프레드시트 자동 생성 · 담당자들에게 권한 공유
교회 구글 드라이브 지정 폴더에 년도_행사이름_부서_v1 형식의 스프레드시트가 자동 생성되고, 담당자 이메일로 편집 권한이 공유됩니다.
▼
3
신청 페이지 링크·큐알코드 공유
생성 결과 화면에 표시되는 링크 또는 큐알코드를 주보·단체 대화방·게시판에 공지합니다.
▼
4
신청자가 폼 작성·제출
신청 기간 동안 참가자·봉사자가 신청 페이지에서 정보를 입력하고 제출합니다. 제출 즉시 스프레드시트에 저장됩니다.
▼
5
행사 담당자가 실시간 현황 확인
담당자는 공유받은 스프레드시트에서 실시간 신청 내역을 확인하고, 필요 시 신청자에게 개별 연락을 취합니다.
▼
6
마감 후 행사 담당자가 결과 확인·보관
마감일이 지나면 신청 페이지가 자동으로 닫힙니다. 행사 담당자는 4자리 비밀번호로 결과를 확인하고, 필요 시 스프레드시트를 다운로드해 보관합니다. 시스템 관리자는 이후 교회 전체 차원에서 백업·정리 작업을 이어갑니다.
핵심 포인트 — 신청 시작일 이후에는 신청서 내용 수정이 불가능합니다. 행사 생성 직후 한 번 더 꼼꼼히 검토하고, 잘못된 부분이 있으면 신청 시작 전에 반드시 수정하세요.
3장

신청자 가이드

교회 행사에 참가하거나 봉사자로 지원하려는 분들을 위한 가이드입니다. 회원가입이나 별도 앱 설치 없이, 공유받은 링크만 열면 곧바로 신청할 수 있습니다.

3-1. 신청 페이지 접속 방법
1
공유받은 링크 주소를 누르기
주보·교회 단체 대화방·게시판 공지 등에서 전달된 링크를 스마트폰 또는 컴퓨터에서 눌러 주세요. 링크를 누르면 해당 행사의 신청 페이지가 바로 열립니다.
2
큐알코드로 접속하기
주보나 포스터에 인쇄된 큐알코드를 스마트폰 카메라로 비추면 자동으로 링크가 인식됩니다. 화면에 뜨는 링크 안내를 누르면 신청 페이지로 이동합니다.
스마트폰이 편합니다 — 신청 페이지는 모바일 화면에 최적화되어 있으며, 링크를 받은 기기에서 그대로 작성·제출하는 것이 가장 빠릅니다.
3-2. 신청 유형 고르기

행사에 따라 한 개 또는 여러 개의 신청 유형이 준비되어 있습니다. 원하는 유형을 눌러 선택한 후 작성을 시작하세요.

신청 유형 예시 설명
참가자 일반 참가를 원하는 분이 선택하는 기본 유형입니다.
봉사자 행사에서 섬김을 원하는 분이 선택하는 유형입니다. 봉사 가능 영역·이전 경험 등을 추가로 묻는 경우가 있습니다.
그 외 자유 유형 행사 특성에 따라 "리더", "초등부 인솔자", "주방팀" 등 별도 유형이 생길 수 있습니다.
정원이 찬 유형은 선택되지 않습니다 — 선착순으로 마감되기 때문에, 정원이 모두 찬 유형은 자동으로 "마감" 표시가 뜹니다. 다른 유형이 열려 있다면 그쪽으로 신청할 수 있습니다.
3-3. 작성 항목

행사에 따라 질문 항목이 달라질 수 있지만, 기본적으로는 아래 항목이 가장 많이 쓰입니다.

항목 작성 안내
이름 교회에서 부르는 이름으로 작성합니다. 같은 이름의 다른 사람과 구별을 위해 성까지 함께 적어 주세요.
전화번호 주 연락 번호를 입력합니다. 급한 공지가 있을 때 문자 또는 전화로 연락받게 됩니다.
이메일 주소 자주 확인하는 이메일을 입력합니다. 행사 안내·준비물·주차 정보 등이 이메일로 전달될 수 있습니다.
생년월일 숙소 배정, 차량 좌석, 식사 준비 등에 참고됩니다. 형식은 폼에 표시된 대로 입력합니다.
성별 숙소 배정 등 필요한 경우에만 사용됩니다.
소속 (부서·사역팀·구역) 소속 부서나 구역을 적습니다. 해당 사항이 없으면 "없음" 또는 "교회등록 준비 중"이라고 적어도 괜찮습니다.
봉사 경험 봉사자 유형을 선택한 경우에만 표시됩니다. 이전 섬김 경력을 간단히 적으면 됩니다. 경험이 없으면 "없음"이라고 적으세요.
추가 질문 행사 성격에 따라 "특이 알레르기", "차량 제공 여부" 등 추가 질문이 나타날 수 있습니다. 해당 칸에 간단히 답변해 주세요.
3-4. 제출하기
1
필수 항목을 모두 채웠는지 확인
빨간 별표(*)가 붙은 항목은 반드시 작성해야 제출 가능합니다. 누락이 있으면 제출 버튼을 눌러도 화면이 움직이지 않고 비어 있는 칸이 강조 표시됩니다.
2
하단의 "제출하기" 버튼 누르기
버튼을 누르면  "신청이 완료되었습니다"라는 안내 화면이 뜹니다. 이 안내가 뜨기 전까지 페이지를 닫거나 뒤로 가기를 누르지 마세요.
3
완료 화면을 확인하고 창 닫기
"신청이 완료되었습니다" 안내가 뜨면 신청이 정상 접수된 것입니다. 별도 확인 이메일은 자동 발송되지 않으니, 완료 화면을 화면 캡처해 두시면 편린합니다.
3-5. 특별 안내 화면들
!

"신청 기간이 아닙니다" 안내가 뜰 때

신청 시작일 이전 또는 마감일 이후에 접속하면 이 안내가 표시됩니다. 주보·공지에 안내된 시작일을 다시 확인하세요. 마감 이후 접속이라면 행사 담당자에게 개별 문의 부탁드립니다.
!

제출 후 작성 내용을 바꾸고 싶을 때

신청자 측에서 제출한 내용을 직접 수정할 수 있는 기능은 없습니다. 행사 담당자에게 연락하여 변경 사항을 알려 주시면 담당자가 스프레드시트에서 해당 내용을 직접 수정합니다.
!

신청을 취소하고 싶을 때

취소 버튼은 없습니다. 취소 의사를 행사 담당자에게 연락 주시면, 담당자가 스프레드시트에서 해당 신청 행을 삭제 처리합니다. 단체 식사·숙소 예약 때문에 가능한 한 마감일 전에 알려 주시면 좋습니다.
4장

행사 담당자 가이드

부서·사역팀의 사역자 또는 팀장이 직접 행사 신청서를 만들고 운영할 수 있도록 안내하는 장입니다. 복잡한 기술 지식 없이도 몇 가지 정보만 입력하면 신청 페이지가 자동으로 만들어지고, 신청 내역은 구글 스프레드시트에 실시간으로 쌓입니다. 별도의 홈페이지 개발이나 행정간사 의뢰는 필요하지 않습니다.

이 장의 핵심 — 담당자는 "만드는 사람"이자 "운영하는 사람"입니다. 플랫폼 접속부터 행사 종료까지 담당자가 스스로 처리하시되, 막히는 부분은 언제든지 시스템 관리자(행정 간사)에게 도움을 요청하세요.
4-1. 플랫폼 접속하기
1
플랫폼 주소 열기
웹브라우저(크롬·사파리·엣지 등)에서 공식 주소(signup.bostononnuri.org)를 여세요. 로그인 화면 없이 곧바로 "신청서 구성" 화면이 열립니다. 회원가입·계정 만들기는 필요하지 않습니다.
2
두 개의 화면 구조 이해하기
상단 버튼을 눌러 두 화면을 오갈 수 있습니다.
· 신청서 구성 화면 — 새 행사를 만드는 곳 (첫 화면)
· 행사 목록 화면 — 지금까지 만들어진 모든 행사를 조회·관리하는 곳
컴퓨터가 더 편합니다 — 입력 항목이 많아서 컴퓨터나 태블릿에서 작업하는 것이 권장됩니다. 스마트폰에서도 동작하지만, 긴 내용 입력에는 불편할 수 있습니다.
4-2. 행사 신청서 만들기

"신청서 구성" 화면에서 아래 항목을 차례대로 채웁니다. 빨간 별표(*)가 붙은 항목은 필수 입력입니다.

입력 항목 작성 안내

4-3. 생성 후 바로 해야 할 일
1
생성 결과 화면 확인
"생성" 버튼을 누른 후 3~5초 기다리면 결과 화면이 뜹니다. 이 화면에는 신청 페이지 링크, 큐알코드 이미지, 스프레드시트로 바로 이동 버튼이 표시됩니다.
2
이메일 확인 — 스프레드시트 공유 알림
입력하신 이메일로 구글 드라이브에서 "공유된 스프레드시트" 알림 메일이 도착합니다. 이메일 내 링크를 누르면 담당자 본인이 해당 스프레드시트를 바로 열어볼 수 있습니다.
이메일이 안 보일 때 — 먼저 스팸함과 "프로모션" 탭을 확인하세요. 발신자는 구글 드라이브 공유 알림 형식입니다. 그래도 없으면 이메일 주소를 잘못 입력했을 가능성이 있으니, 행사 목록에서 수정 기능으로 담당자 이메일을 바로잡거나 시스템 관리자에게 문의해 주세요.
3
링크 복사 후 공지 채널에 배포
링크 옆의 복사 버튼을 눌러 주소를 클립보드에 복사한 다음, 공지 채널에 공유합니다.
4
큐알코드 이미지 저장
큐알코드 영역을 오른쪽 클릭(또는 모바일에서 길게 누르기)하면 이미지를 저장할 수 있습니다. 저장된 이미지는 주보·포스터·대화방에 첨부해 사용하세요.
4-4. 신청 현황 확인 (스프레드시트)

이메일로 공유받은 스프레드시트를 열면 신청 유형마다 별도 탭(시트)이 자동으로 생성되어 있습니다. 각 탭에 해당 유형의 신청자 목록이 제출 순서대로 한 줄씩 쌓입니다.

열(컬럼) 이름 설명
제출시각 (동부 시간) 신청자가 제출 버튼을 누른 시각입니다. 보스턴 동부 시간 기준으로 자동 기록됩니다.
이름 신청자가 입력한 이름입니다.
카테고리 신청 유형(참가자·봉사자 등)이 자동으로 표시됩니다.
전화번호 · 이메일 신청자가 입력한 연락처입니다. 안내 문자·이메일 발송에 활용합니다.
생년월일 · 성별 · 소속 행사 준비(숙소 배정·차량·식사)에 참고할 수 있는 기본 정보입니다.
봉사경험 · 추가질문 유형·행사 성격에 따라 표시되는 추가 답변입니다. 빈 칸이면 해당 유형에 그 질문이 없었다는 뜻입니다.
4-5. 담당자가 스프레드시트에서 할 수 있는 일
가

실시간 신청 현황 확인

스프레드시트는 계속 업데이트되므로 필요할 때마다 열어 최신 현황을 확인할 수 있습니다. 새로고침 또는 페이지 재접속을 통해 가장 최근 데이터를 볼 수 있습니다.
나

신청자 데이터 직접 수정

신청자가 제출한 정보에 오타가 있거나 내용이 바뀐 경우, 해당 칸을 직접 클릭하여 수정할 수 있습니다. 단, 제출시각 열(첫 번째 열)은 수정하지 마세요. 저장 순서 관리에 쓰입니다.
다

신청 취소·중복 행 삭제

신청자가 취소를 요청하거나 실수로 두 번 제출한 경우, 해당 행(가로 줄) 번호를 오른쪽 클릭하여 "행 삭제"를 선택하면 지워집니다.
라

스프레드시트 기능 자유롭게 활용

정렬·필터·색상 채우기·조건부 서식 등 일반적인 구글 스프레드시트 기능은 모두 사용 가능합니다. 행사 준비에 맞게 편하게 활용하세요. 단, 시트 탭 이름 변경·삭제는 하지 마세요. 데이터 저장에 문제가 생길 수 있습니다.
주의 — 스프레드시트의 첫 행(헤더)과 기존 열 구조는 절대 변경하지 마세요. 새 신청이 들어올 때 잘못된 칸에 값이 들어갈 수 있습니다. 꼭 필요하면 시스템 관리자에게 먼저 문의해 주세요.
4-6. 기존 행사 관리 (행사 목록 화면)

플랫폼 상단의 "행사 목록" 버튼을 누르면 지금까지 생성된 모든 행사가 나열됩니다. 본인이 만든 행사는 물론, 다른 담당자가 만든 행사도 목록에는 표시되지만 수정·결과 보기·삭제는 4자리 비밀번호를 알고 있는 사람만 가능합니다.

버튼 역할 · 비밀번호 필요 여부
링크 복사 해당 행사의 신청 페이지 주소를 클립보드에 복사합니다. 비밀번호 없이 누구나 누를 수 있습니다.
결과 보기 현재까지 접수된 신청자 목록을 화면에서 바로 조회합니다. 4자리 비밀번호 입력 필요.
수정 행사 이름·날짜·유형 등 설정을 수정합니다. 4자리 비밀번호 확인 후만 진입 가능하며, 신청 시작일 이후에는 수정할 수 없습니다. 링크 주소는 그대로 유지됩니다.
삭제 행사 목록에서 제거합니다. 4자리 비밀번호 확인 후에만 실행되며, 스프레드시트 자체는 구글 드라이브에 그대로 남아 있습니다.
"수정"은 신청 시작 전까지만 — 시작일이 지난 행사는 수정할 수 없습니다. 이미 접수된 데이터 구조가 바뀌면 기존 신청자 데이터가 어긋날 위험이 있기 때문입니다. 꼭 바꿔야 하는 경우에는 기존 행사는 그대로 두고 새 행사를 하나 더 만들어 새 링크로 공지하는 방법이 안전합니다.
4-7. 신청자에게 안내 연락하기

스프레드시트의 이메일 열·전화번호 열을 활용하면 단체 안내를 보낼 수 있습니다. 자주 쓰는 방법은 다음과 같습니다.

  • 이메일 단체 발송: 이메일 열을 복사한 뒤 본인 메일의 "숨은 참조" 칸에 붙여넣어 서로의 주소를 가리고 보냅니다.
  • 문자 리마인더: 일정·준비물·주차 안내처럼 간단한 공지는 전화번호로 문자를 보내는 것이 효과적입니다.
  • 대기 명단 관리: 시트에 탭 하나를 추가하여 정원 초과자를 별도로 관리할 수 있습니다.
4-8. 문제가 생겼을 때

아래 상황이 발생하면 시스템 관리자(행정 간사)에게 문의하세요. 

  • 행사 생성 시 지정한 4자리 비밀번호를 잊어버렸을 때
  • 담당자 이메일을 잘못 입력해 스프레드시트 공유를 못 받았을 때
  • 신청 시작 후 꼭 바꿔야 할 오류를 발견했을 때 (수정 불가 상황)
  • 스프레드시트 구조(헤더·탭)가 손상되어 새 신청이 엉뚱한 칸에 저장될 때
  • 플랫폼 자체가 접속되지 않거나, "저장 중"에서 계속 멈추는 등 기술적 이상 징후가 보일 때
  • 처음 사용하시는 경우 생성 대행을 요청하고 싶을 때 (담당자 직접 만드는 것이 권장되지만, 필요 시 지원 가능)
5장

시스템 관리자 가이드

이 플랫폼을 뒷받침하는 행정 간사를 위한 안내입니다. 각 사역팀의 행사 담당자가 스스로 신청서를 만들어 쓰는 구조이기 때문에, 시스템 관리자의 주된 역할은 운영·백업·지원입니다. 행사 생성 자체를 시스템 관리자가 도맡는 것이 아니라, 뒤에서 플랫폼이 안정적으로 돌아가고 데이터가 잘 쌓이도록 살피는 것이 목적입니다.

5-1. 시스템 관리자의 4가지 역할
1
행사 담당자 지원
비밀번호 분실·이메일 오류·구조적 오류 등으로 담당자가 스스로 해결하기 어려운 상황에서 관리자 권한으로 직접 처리해 드립니다.
2
생성된 신청서 전체 관리
교회 전체 행사 신청서가 쌓여 있는 구글 드라이브 지정 폴더를 주기적으로 점검하고, 너무 오래된 행사는 정리·아카이브합니다.
3
데이터 백업
구글 드라이브 자체가 영구 보관하지만, 중요 행사는 엑셀 파일로 별도 백업하여 외부 저장소(교회 컴퓨터·외장 하드 등)에 이중으로 보관합니다.
4
시스템 유지관리
플랫폼이 정상 동작하는지 점검하고, 필요 시 업데이트·배포를 진행합니다. 기술적인 세부 사항은 별도의 내부 기술 문서를 참고합니다.
행사 생성은 담당자가 직접 하는 것이 권장 — 시스템 관리자도 신청서를 생성할 수는 있지만, 각 부서·사역팀의 상황을 가장 잘 아는 것은 담당자 본인입니다. 요청이 있을 때만 대신 만들고, 가능하면 담당자가 직접 만들어보도록 안내해 주세요. 처음 사용 시 함께 앉아 한 번만 따라 만들면 다음부터는 혼자 할 수 있게 됩니다.
5-2. 행사 담당자 지원하기

담당자에게서 자주 들어오는 요청 유형과 대응 방법입니다.

가

"4자리 비밀번호를 잊었어요"

수정·삭제·비밀번호 재설정이 필요한 경우에는 시스템 관리자에게 문의 바랍니다.
나

"스프레드시트 공유 이메일이 안 왔어요"

먼저 담당자에게 스팸함과 "프로모션" 탭 확인을 안내합니다. 그래도 못 찾으면 행사 목록에서 수정 기능을 열어 담당자 이메일을 다시 정확히 입력합니다. 저장 시 다시 공유 알림이 발송됩니다. 이메일 주소 자체가 틀린 경우에는 담당자에게 정확한 주소를 다시 받아 적용합니다.
다

"신청 시작 후에 오류가 발견됐어요"

플랫폼 자체로는 수정이 불가능하므로, ① 기존 행사는 그대로 두고 새 행사를 하나 더 생성해 새 링크로 교체 공지하거나, 작은 오류라면 ② 스프레드시트 상에서 수동 보정하는 두 가지 방법 중 담당자와 상의해 결정합니다.
라

"처음이라 만드는 법을 잘 모르겠어요"

함께 앉아 한 번만 따라 만들어보도록 안내하는 것이 가장 효과적입니다. 이 가이드 4장 "행사 담당자 가이드"를 같이 보면서 진행하면 됩니다. 시간이 급한 경우 시스템 관리자가 대신 생성해 주되, 다음번에는 담당자가 직접 할 수 있도록 과정을 공유해 두는 것이 좋습니다.
5-3. 데이터 백업·보관
1
구글 드라이브 자동 보관 (기본)
모든 행사의 스프레드시트는 교회 지정 구글 드라이브 폴더에 자동 저장되며, 별도 백업 없이도 영구 보관됩니다. 구글 계정이 해지되지 않는 한 데이터는 사라지지 않습니다.
2
중요 행사는 수동 이중 백업
장기 수련회·큰 규모 모임처럼 개인정보 보존 책임이 큰 행사는 엑셀 파일(또는 쉼표 구분 문서)로 다운로드하여 교회 업무용 저장소에 별도 보관합니다. 구글 스프레드시트 → 파일 메뉴 → 다운로드 → 원하는 형식 선택.
3
오래된 행사 정리
행사 목록이 지나치게 길어지면 담당자들이 최근 행사를 찾기 어려워집니다. 종료된 지 일정 기간이 지난 행사는 구글 드라이브에서 스프레드시트를 직접 열어 결과를 백업한 뒤 목록에서 삭제합니다(스프레드시트 원본은 드라이브에 그대로 유지됨).
개인정보 취급 — 다운로드한 파일에는 신청자 이름·연락처·이메일 등이 포함되어 있습니다. 교회 업무 목적 외 공유 금지이며, 내부 규정에 따라 일정 기간 후 파기하는 것이 바람직합니다.
5-4. 시스템 유지관리

플랫폼 자체가 정상 동작하도록 점검하는 업무입니다. 기술적 세부는 내부 기술 문서(프로젝트 지식 문서)를 함께 참고하세요.

점검 항목 주기 · 방법
플랫폼 접속 확인 월 1회 이상 공식 주소에 접속하여 화면이 정상적으로 뜨는지 확인합니다.
테스트 신청서 생성 분기마다 테스트용 가짜 행사를 만들어 전체 흐름(생성→제출→저장→삭제)이 정상인지 확인합니다.
업데이트 배포 플랫폼 수정(index.html) 시 깃허브 저장소에 업로드하면 Vercel이 자동으로 반영합니다. 백엔드(Code.gs) 수정 시에는 반드시 "새 버전으로 배포"를 실행해야 반영됩니다.
사용량 모니터링 구글 앱스 스크립트의 무료 사용 한도(하루 실행 시간·호출 수)를 넘지 않는지 주기적으로 확인합니다.
6장

자주 묻는 질문

사용 중 자주 묻는 질문을 정리했습니다. 더 궁금한 점이 있으면 행정 간사에게 문의해 주세요.

Q
(담당자용) 행사 신청서를 꼭 제가 직접 만들어야 하나요? 행정 간사가 대신 만들어 주시면 안 되나요?
이 플랫폼은 각 사역팀 담당자가 직접 만들어 쓰도록 설계된 도구입니다. 각 부서·사역팀의 상황(모집 유형·정원·추가 질문 등)을 가장 잘 아는 분이 담당자 본인이기 때문입니다. 처음 한두 번은 4장을 보면서 따라 만들어 보시고, 막히는 부분은 언제든 시스템 관리자에게 문의하세요. 시간이 급한 경우 시스템 관리자가 대신 만들어 드릴 수도 있지만, 가능하면 직접 만들어보시는 것을 권장합니다.
Q
(담당자용) 행정 간사가 없으면 아예 혼자서 처음부터 끝까지 다 해낼 수 있나요?
네, 담당자 혼자서 행사 생성 → 링크 공유 → 신청자 관리 → 결과 확인까지 모든 과정을 수행할 수 있도록 만들어져 있습니다. 시스템 관리자는 비밀번호 분실·이메일 오류 같은 예외 상황에서만 지원하는 역할이며, 평소 운영은 담당자 주도로 진행됩니다.
Q
신청 페이지에 접속했는데 "신청 기간이 아닙니다"라고만 뜹니다.
신청 시작일이 아직 안 되었거나, 이미 마감일이 지난 경우입니다. 주보·공지의 시작일을 다시 확인해 주세요. 이미 마감 이후라면 행사 담당자에게 직접 문의하는 것이 가장 빠릅니다.
Q
신청 완료 이메일이 안 왔어요. 제대로 접수된 건가요?
이 플랫폼은 신청자에게 자동 확인 이메일을 보내지 않습니다. 제출 직후 "신청이 완료되었습니다" 안내 화면이 떴다면 정상적으로 접수된 것입니다. 안심되려면 완료 화면을 화면 캡처해 보관해 두시거나, 담당자에게 확인을 요청하세요.
Q
제출 후 내용을 바꾸고 싶은데 어떻게 하나요?
신청자가 직접 수정할 수 있는 기능은 없습니다. 행사 담당자에게 변경 내용을 알려 주세요. 담당자가 스프레드시트에서 해당 항목을 직접 고칠 수 있습니다.
Q
같은 사람이 실수로 두 번 제출했어요. 어떻게 하나요?
행사 담당자가 스프레드시트에서 중복된 행(가로 줄)을 직접 삭제할 수 있습니다. 담당자에게 연락하여 "중복 제출이 있으니 삭제 부탁드립니다"라고 요청해 주세요.
Q
결과 보기 버튼을 눌렀는데 비밀번호가 기억나지 않아요.
행사 생성 시 지정한 4자리 비밀번호를 분실했다면, 행정 간사(시스템 관리자)에게 문의하세요. 관리자가 별도의 안전한 절차로 결과 확인이나 수정·삭제 작업을 도와드립니다.
Q
담당자인데 스프레드시트 공유 이메일이 안 왔어요.
먼저 스팸함과 "프로모션" 탭을 확인해 주세요. 발신자는 보스턴온누리 또는 구글 드라이브 공유 알림 형식입니다. 그래도 없으면 시스템 관리자에게 연락하여 이메일 주소가 정확히 입력되었는지 확인하고, 공유를 재요청하면 됩니다.
Q
행사 정보(날짜·제목 등)를 수정하고 싶은데 수정 버튼이 안 눌려요.
신청 시작일이 이미 지난 행사는 수정할 수 없습니다. 이미 접수된 데이터의 구조가 바뀌면 기존 신청자 데이터가 어긋날 위험이 있기 때문입니다. 꼭 변경해야 한다면 기존 행사는 그대로 두고 새 행사를 하나 더 만들어, 공지에서 새 링크로 안내하는 방법을 권장합니다.
Q
큐알코드를 인쇄했는데 한 번 만든 후 주소가 바뀔 일이 있나요?
한 번 생성된 신청 페이지의 링크 주소는 수정·삭제 전까지 영구히 같은 값입니다. "수정"으로 설정을 바꿔도 링크 자체는 유지되므로 인쇄물을 다시 출력할 필요가 없습니다.
Q
신청자 개인정보는 안전하게 관리되나요?
신청 내용은 교회 공식 구글 드라이브에 저장되며, 스프레드시트는 행사 담당자와 시스템 관리자에게만 공유됩니다. 신청자의 이름·연락처는 해당 행사 업무 목적 외에 사용되지 않으며, 외부로 유출되지 않습니다. 행사 종료 후에는 교회 개인정보 보호 방침에 따라 안전하게 관리·파기됩니다.
Q
모바일(휴대폰)에서도 문제없이 쓸 수 있나요?
네, 신청 페이지는 모바일 화면에 최적화되어 있습니다. 대부분의 신청자는 스마트폰으로 링크를 받고 그 자리에서 작성·제출합니다. 담당자가 행사 신청서를 만들고 관리하는 화면 역시 모바일에서 동작하지만, 입력 항목이 많아 컴퓨터나 태블릿에서 작업하는 것이 편합니다.
Q
제출이 "저장 중..." 상태에서 멈춰요.
드물게 첫 제출 시 서버가 깨어나는 데 2~4초 걸릴 수 있습니다. 잠시만 기다려 주세요. 5초 이상 멈춰 있다면, 인터넷 연결을 확인한 후 페이지를 새로고침하고 다시 제출합니다. 그래도 안 되면 행사 담당자에게 문의해 주세요.
Q
행사 생성 후 스프레드시트 파일명을 바꿔도 되나요?
파일명은 바꾸지 마세요. 이 플랫폼은 파일명으로 저장 위치를 찾으므로, 이름이 바뀌면 새 신청이 들어왔을 때 제대로 저장되지 않을 수 있습니다. 탭(시트) 이름도 그대로 두세요.
